Réalisations


8. SAÉ 4.A.02.1 : Développement d'un site web REST

  • Structuration

    • Cette saé portait sur réaliser un site web d'une pizzeria 'Pizzeria Mazzorello' respectant l'architecture REST. Il devait donc y'avoir plusieurs end-points afin de permettre à des utilisateurs externes de récupérer les données et de les modifier dans la base de données. Les end-points étaient les suivants :

    • Ingrédients : id, nom, prix
    • Pizzas : id, nom, type de pâte, ingrédient, prix de base
    • Commandes : id, client, date,

  • Sécurité

    • Mise en place de différents système d'authentification notamment API Token et JWT token.

    image placeholder

    Compétences et savoir-faire acquis

    • Architecture REST : Nous ne connaisions pas les architectures REST, ce projet nous à permis de les comprendre et d'apprendre à créer des site la respectant.
    • Postgresql : Pour ce projet, l'utilisation d'une base de données était nécessaire.

    Savoir-êtres acquis

    • Autonomie : Ce projet est le seul que nous avons eu à réaliser entièrement seul mais étant très autonome, cela ne m'a pas posé de problèmes.
    • Respect des délais : Les délais pour ce projet étaient assez court, ce fut un bon exercice pour bien faire attention à rendre dans les temps tout en respectant le cachier des charges

9. SAÉ 4.A.02.2 : Développement d'une application complexe

  • Idées de gameplay

    • Nous avons reçu un cahier des charges, avec toutes les fonctionnalités minimales que devrais avoir l'application finale. Notre première mission était donc de réfléchir à des éléments de gameplay et à un thème permettant de répondre aux demandes. Nous nous sommes d'abord concentré sur les éléments de gameplay, en fournissant deux versions prototypes sans aucun thèmes ou élément audio visuel, uniquement une base de gameplay et des formes géométriques pour répresenter les élements du jeu.

  • Multijoueurs avec node.js et socket.io

    • Durant cette saé, les professeurs ont particulièrement insisté sur la pertinence des tests et de la documentation que nous devons fournir. Ce projet fut donc l'occasion de nous apprendre à faire des tests corrects, pertinent mais aussi à nous introduire aux outils d'analyses de code tel que CodeSonar.

    Compétences et savoir-faire acquis

    • JavaScript ES6 : Nous n'avions jamais codé en javascript avant ce projet. Ce fut l'occasion d'apprendre et de mettre en pratique ces nouvelles compétences.
    • Nodes.js, socket.io : Grâce à socket.io, nous avons pu créer un jeu multijoueur où plusieurs aprties peuevnt se dérouler en même temps grâce à l'utilisation des rooms.
    • Babel, Prettier : gérer des dépendances (package.json) dans un projet js.
    • Canvas : dessiner des formes, des images dans un canvas

    Savoir-êtres acquis

    • Travail d'équipe : Nous avons réalisé ce projet en binôme, qui était prévu d'être fait en trinôme, ce qui à mis à l'épreuve de finir dans les temps avec un tiers de notre force de travail.
    • Collaboration : Afin de pouvoir finir dans les temps, une bonne répartition du travail était essentielle à cause de cet handicap. Nous avons relevé ce défi haut la main, en respectant le cahier des charges et en ayant même plus des fonctionnalités suplémentaires.
    • Qualité des dépôts git (Commits,Milestones) : le professeur à particulièrement insisté sur la qualité de notre dépôt git et sur la gestion des tâches, nous avons donc appris à mieux utiliser les outils collaboratif de GitLab et avons acquis une meilleure éthique de travail.
